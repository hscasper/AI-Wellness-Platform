       ```javascript
       module.exports = {
         getItemAsync: jest.fn().mockResolvedValue(null),
         setItemAsync: jest.fn().mockResolvedValue(undefined),
         deleteItemAsync: jest.fn().mockResolvedValue(undefined),
       };
       ```

    4. Create `frontend/__mocks__/@react-native-async-storage/async-storage.js`:
       ```javascript
       const store = {};
       module.exports = {
         getItem: jest.fn((key) => Promise.resolve(store[key] || null)),
         setItem: jest.fn((key, value) => { store[key] = value; return Promise.resolve(); }),
         removeItem: jest.fn((key) => { delete store[key]; return Promise.resolve(); }),
         clear: jest.fn(() => { Object.keys(store).forEach(k => delete store[k]); return Promise.resolve(); }),
       };
       ```

    5. Verify configuration by running `cd frontend && npx jest --watchAll=false --passWithNoTests`. It should exit 0 with "No tests found" or similar.

    NOTE: If `jest-expo` has peer dependency issues with the installed Expo version, check the error messages and install the compatible version. The key is that `npm test` exits 0 without configuration errors.
